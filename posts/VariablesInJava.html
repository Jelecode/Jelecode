Variables in the Java programming language

Variables are containers in which values are stored. A single variable during Java program execution can have different values and that is why it is called a variable. That means that the value of one variable can be changed at any moment.

It is possible to change the value of a variable any number of times, as long as its type is respected. 

    String message = "Helo World";
    message = 33;

This code is incorrect and produces an error because a String variable cannot accept an int data.

The variable consist of at least :

- the data type that the variable will be able to accept

- an identifier representing the name of the variable

Declaration and initialization

Declaration of a variable is the process of its creation. Assigning a data type and name to a variable. 

    String - data type
    message - identifier

Initialization is the process of assigning a value to a variable

Declaration and initialization can be done separately or in one command : 

    String message = "How are you?";

Variable naming

- the variable name can be composed of letters, numbers, and the underscore and dollar characters

- the variable name must have at least one character

- a variable name must not start with a number

- the variable name is case sensitiv

- the variable name must not contain spaces

- Java keywords cannot be used as a variable name

Good variable naming practice : 

String userName;

int userNumber;

float birthYear;

doouble firstOperand;

Variables without an explicitly defined type

To create a variable without explicitly defining its type, the keyword ( var ) is used. 

var name = "Marco";

A variable without an explicitly defined type must be declared and initialized in a single statment.

Since the Java compiler relies on values to guess the type of a variable, using the word var in uninitialized statements is not possible because the compiler cannot guess which data type it belongs to.
